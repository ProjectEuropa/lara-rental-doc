      # Laravel on レンタルサーバーの使い方

      # ディレクトリ構造

      Laravel-Project
├── .htaccess #Laravelのドキュメントルートを無理やり変えるための設定ファイル
├── .env #Laravelアプリケーションの環境設定ファイル
├── app #Laravelアプリケーションのメインディレクトリ
├── artisan #基本編集しない(artisan コマンド設定ファイル)
├── bootstrap #基本編集しない(Laravel設定ディレクトリ)
├── composer.json #composer設置ファイル
├── composer.lock #composer設置ファイル
├── config #Laravel全般の設定ディレクトリ
├── database #データベース設定関連ディレクトリ
├── package.json #基本編集しない
├── phpunit.xml #基本編集しない
├── public #公開ディレクトリ
├── resources #view関連で設定するディレクトリ
├── routes #ルーティング設定ディレクトリ
├── server.php #基本編集しない
├── storage #基本編集しない
├── tests #基本編集しない(テストコード関連ディレクトリ)
├── vendor #基本編集しない
└── webpack.mix.js #基本編集しない(Node.js実行時には設定が必要です)

      # .htaccessの設定

      <IfModule mod_rewrite.c>
    <IfModule mod_negotiation.c>
        Options -MultiViews
    </IfModule>

    RewriteEngine on
    RewriteBase /
    RewriteRule ^$ public/ [L]
    RewriteRule ^(.+)$ public/$1 [L]
</IfModule>

      # レンタルサーバー特有のLaravelの使い方(SSHの場合)

      # DB接続方法

      • .envを編集し、レンタルサーバーでのMySQLでの接続情報にしたがって記述してください
      DB_CONNECTION=mysql #固定値
DB_HOST=127.0.0.1 #MySQLのHOSTアドレス。特に記載がなければこのまま
DB_PORT=3306 #DBport。特に記載がなければこのまま
DB_DATABASE=laravel #DB名。MySQLの接続情報にしたがって記述してください
DB_USERNAME=root #DB接続ユーザー名。MySQLでの接続情報にしたがって記述してください
DB_PASSWORD= #DB接続パスワード。MySQLでの接続情報にしたがって記述してください
      • .env編集後php aritsan migrateでマイグレーションを実行します。
      $ php artisan migrate

      # レンタルサーバー特有のLaravelの使い方(FTPの場合)

      普通のLaravelの使い方であれば、検索すれば出てくるので割愛します。 ここではレンタルサーバーでのLaravelの使い方にフォーカスを当てます。

      # DB接続方法

      • .envを編集し、レンタルサーバーでのMySQLでの接続情報にしたがって記述してください
      DB_CONNECTION=mysql #固定値
DB_HOST=127.0.0.1 #MySQLのHOSTアドレス。特に記載がなければこのまま
DB_PORT=3306 #DBport。特に記載がなければこのまま
DB_DATABASE=laravel #DB名。MySQLの接続情報にしたがって記述してください
DB_USERNAME=root #DB接続ユーザー名。MySQLでの接続情報にしたがって記述してください
DB_PASSWORD= #DB接続パスワード。MySQLでの接続情報にしたがって記述してください

      # マイグレーション手順

      • ローカル環境でマイグレーションファイル作成
      • ローカル環境でマイグレーション実行
      • phpMyAdminでローカル環境のデータをエクスポート
      • phpMyAdminでローカル環境からエクスポートしたデータをレンタルサーバー上のMySQLにインポート

      # デプロイ手順

      • ローカルでcomposerコマンドを叩いて、composer.jsonが変更された時は、vendorディレクトリを全てアップロードし直します
      • ローカルで編集したファイルを全てアップロードし直します
